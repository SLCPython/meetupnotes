<h2> Testing in Python </h2>
Ruminations on testing in general and python specifically

https://github.com/eigenholser/jpeg_rename

http://lab.hakim.se/reveal-js/#/ - tool Scott used to make his presentation

Unit testing 
* regression protection
* helps to fill out what you need to build

types:
* unit testing: testing units of code; making sure they function as designed
* integration: across layers of code 
* code coverage: measure how much of your code is covered by tests
* doctests: put tests inside your docstring

python tools:
* unittest
* pytest
* nose
* doctest
* mock (part of the standard library in Python 3.4)

Approaches to testing:
* test first
* test last
* test-driven development
-> use all, just test-last will be hurting

structure of a test:
* arrange (setup)
* act (call code to test)
* assert (check results)
write backwards


On command line:
pip install coverage # <- for code coverage reporting

In code, to tell pytest to ignore a block for coverage:
# pragma : no cover 

isolate your tests to avoid breakage from unrelated code changes

parametrized tests: cover multiple paths in your code with one test

doctests (command line or script):
python -m doctest -v filename.py?

Appendix A: Docstrings
Have built in documentation in python

def myfunction (a,b):
    """ Doc string for my function. Any text can go here.
    
    Parameters
    a : float
    b : float
    
    Returns
    c : float
        the sum 
    """
    return a+b

>>> help(myfunction) # this will give the string I defined, i.e. "Doc string for my function..."
The string is stored in a hidden variable
>>> myfunction.__docstring__
